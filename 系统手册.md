# 系统手册 · pp-zero-lab

> 分布式训练显存与吞吐优化实验平台 —— 部署指南 + 接口说明
>
> 平台目标：在小型 GPT 上实测并对比 **流水线并行（GPipe / 1F1B）** 与 **ZeRO 显存优化（Stage 1/2/3）**，产出标准化指标 `results/metrics.csv` 供可视化与报告使用。
>
> 代码仓库：https://github.com/yuese12333/pp-zero-lab.git

---

## 快速开始

已配置好 WSL2 + conda + PyTorch + DeepSpeed 环境的用户，三步即可复现核心实验：

```bash
git clone https://github.com/yuese12333/pp-zero-lab.git
cd pp-zero-lab
bash scripts/run_all.sh && python src/theory.py
```

可选扩展实验（扫描曲线、组合建模，见 §三）：

```bash
python src/sweep.py
python src/combo.py
```

也可以使用部署脚本完成环境检查、数据生成和可视化启动：

```bash
# 检查 Python / PyTorch / CUDA / DeepSpeed 可见性
bash scripts/deploy.sh check

# 核心实测 + ZeRO 多卡理论值
bash scripts/deploy.sh core

# 扩展数据：sweep.csv / combo.csv / tradeoff.csv
bash scripts/deploy.sh extended

# 启动可视化系统
bash scripts/deploy.sh viz
```

首次配置环境请见下方完整部署指南。

## 一、环境要求

| 项目 | 要求 / 实测环境 |
|------|------|
| 操作系统 | Windows 10/11 + **WSL2 (Ubuntu)**（DeepSpeed 在原生 Windows 难以编译，必须用 WSL2 或 Linux） |
| GPU | NVIDIA 显卡，支持 CUDA。实测 RTX 3060 Laptop（6GB） |
| NVIDIA 驱动 | 支持 CUDA 12.x（实测驱动 581.08 / CUDA 13.0，向下兼容） |
| Python | 3.10（conda 环境 `ppzero`） |
| PyTorch | 2.5.1 + cu121（实测；仓库 `requirements.txt` 声明 `>=2.0.0`） |
| CUDA Toolkit | 12.1（可选；本项目 ZeRO 已用 PyTorch AdamW + `torch_adam: true`，通常无需 JIT 编译 fused op） |
| DeepSpeed | 0.19.2（实测；仓库声明 `>=0.12.0`） |

> **显存说明**：本平台默认模型约 30M 参数，单卡峰值显存 < 3GB，6GB 显卡即可运行。更大显存可调大 `--n-layer / --n-embd / --batch-size`。

## 二、部署指南

### 1. 启用 WSL2（Windows 侧，管理员 PowerShell）

```powershell
wsl --install -d Ubuntu
```

安装后设置 Linux 用户名与密码。进入 WSL 后验证 GPU 直通成功（无需在 Linux 内安装显卡驱动，复用 Windows 驱动）：

```bash
nvidia-smi   # 能看到显卡型号即成功
```

> **常见问题**：若启动报 `Wsl/Service/E_UNEXPECTED`，常见原因为：①安全软件（如火绒）的 ARP 防护拦截 WSL 虚拟网卡 → 放行或临时关闭；②WSL 本体升级残留损坏 → 卸载商店版 WSL 后从 [WSL Releases](https://github.com/microsoft/WSL/releases) 重新安装 MSI；③localhost 代理与 NAT 冲突 → 在 `C:\Users\<用户>\.wslconfig` 写入 `[wsl2]` / `networkingMode=mirrored`。

### 2. 安装 Miniconda（WSL 内）

```bash
cd ~
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh   # 一路 yes，init 选 yes
# 重开终端，提示符出现 (base)
```

### 3. 创建并激活环境

```bash
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
conda create -n ppzero python=3.10 -y
conda activate ppzero
```

### 4. 安装依赖

```bash
# PyTorch（cu121，兼容 CUDA 12.x / 13.x 驱动）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# 项目其余依赖（deepspeed / numpy / pandas）
pip install -r requirements.txt
```

也可以使用部署脚本安装依赖：

```bash
bash scripts/deploy.sh install
```

如果系统中已安装合适版本的 PyTorch，只想安装 `requirements.txt` 中的其余依赖：

```bash
SKIP_TORCH=1 bash scripts/deploy.sh install
```

若 DeepSpeed 初始化仍报缺少 `nvcc`，可额外安装 CUDA Toolkit 并设置 `CUDA_HOME`：

```bash
conda install -c "nvidia/label/cuda-12.1.0" cuda-toolkit -y
mkdir -p $CONDA_PREFIX/etc/conda/activate.d
echo 'export CUDA_HOME=$CONDA_PREFIX' > $CONDA_PREFIX/etc/conda/activate.d/cuda.sh
echo 'export PATH=$CUDA_HOME/bin:$PATH' >> $CONDA_PREFIX/etc/conda/activate.d/cuda.sh
```

### 5. 验证环境

```bash
# PyTorch 能用 GPU（应输出 True 和显卡名）
python -c "import torch; print(torch.__version__, torch.cuda.is_available(), torch.cuda.get_device_name(0))"

# DeepSpeed 可导入
python -c "import deepspeed; print(deepspeed.__version__)"
```

等价的脚本检查命令：

```bash
bash scripts/deploy.sh check
```

该命令会输出 Python 版本、`torch / numpy / pandas / deepspeed` 是否可导入、CUDA 是否可用，以及 `deepspeed` 命令是否在 `PATH` 中。若 CUDA 或 DeepSpeed 不可用，`run_all.sh` 会跳过 ZeRO 单卡实测行，但仍会运行 baseline、流水线仿真和理论建模。

### 6. 拉取项目代码

在 WSL **原生目录**（如 `~`）下克隆仓库，而非 `/mnt/<盘符>/`，否则文件 IO 极慢，严重拖累训练：

```bash
cd ~
git clone https://github.com/yuese12333/pp-zero-lab.git
cd pp-zero-lab
```

> 若未安装 git：`sudo apt update && sudo apt install -y git`

用 VS Code / Cursor 远程编辑（自动连接 WSL，编辑 Linux 侧文件）：

```bash
code .
```

> **Windows ↔ WSL 同步**：若在 Windows 侧（如 `E:\pp-zero-lab`）改代码，需同步到 WSL 再跑实验：
> `rsync -av --exclude '__pycache__' /mnt/e/pp-zero-lab/ ~/pp-zero-lab/`

## 三、运行说明

### 3.1 部署脚本总览

项目提供 `scripts/deploy.sh` 作为统一部署入口。它不会改变 CSV 字段契约，只负责把常用动作标准化。

| 命令 | 作用 | 写入 / 服务 |
|------|------|-------------|
| `bash scripts/deploy.sh check` | 检查 Python、关键包、CUDA、DeepSpeed | 只读 |
| `bash scripts/deploy.sh install` | 安装 PyTorch CUDA wheel 与 `requirements.txt` | Python 环境 |
| `bash scripts/deploy.sh core` | 跑核心实测，并追加 ZeRO 多卡理论值 | 覆盖 `metrics.csv` 后追加理论行 |
| `bash scripts/deploy.sh extended` | 重新生成扩展数据 | 覆盖 `sweep.csv` / `combo.csv` / `tradeoff.csv` |
| `bash scripts/deploy.sh full` | 执行 `check` + `core` + `extended` | 生成全部 CSV |
| `bash scripts/deploy.sh viz` | 启动本地可视化服务 | `http://127.0.0.1:8765/viz/index.html` |
| `bash scripts/deploy.sh status` | 查看 CSV 行数与可视化产物 | 只读 |

部署脚本支持以下环境变量：

| 变量 | 默认 | 说明 |
|------|------|------|
| `PYTHON` | `python` | Python 解释器，例如 `PYTHON=python3` |
| `STEPS` | `20` | 传给 `run_all.sh` 的训练步数 |
| `WARMUP` | `2` | 传给 `run_all.sh` 的预热步数 |
| `BATCH_SIZE` | `4` | 传给 `run_all.sh` 的 batch size |
| `HOST` | `127.0.0.1` | 可视化服务监听地址 |
| `PORT` | `8765` | 可视化服务端口 |
| `SKIP_TORCH` | `0` | `install` 时设为 `1` 可跳过 PyTorch 安装 |

示例：

```bash
# 用 python3 运行，减少训练步数做快速冒烟测试
PYTHON=python3 STEPS=3 WARMUP=1 bash scripts/deploy.sh core

# 启动 9000 端口的可视化服务
PORT=9000 bash scripts/deploy.sh viz
```

### 单个实验

```bash
cd ~/pp-zero-lab
conda activate ppzero

# Baseline（单卡全量，无 ZeRO / 无分布式）
python src/train.py --config baseline --steps 20 --batch-size 4

# 流水线并行仿真（纯 PyTorch，不依赖 DeepSpeed）
python src/train.py --config gpipe --steps 20 --batch-size 4 --num-stages 4 --micro-batches 8
python src/train.py --config 1f1b  --steps 20 --batch-size 4 --num-stages 4 --micro-batches 8

# ZeRO Stage 1/2/3（DeepSpeed 启动；--deepspeed 为开关，json 由 --config 自动映射）
deepspeed --num_gpus=1 src/train.py --config zero1 --steps 20 --batch-size 4 --deepspeed
deepspeed --num_gpus=1 src/train.py --config zero2 --steps 20 --batch-size 4 --deepspeed
deepspeed --num_gpus=1 src/train.py --config zero3 --steps 20 --batch-size 4 --deepspeed
```

> **注意 1**：ZeRO 配置 `configs/zeroN.json` 的 `train_batch_size` 必须等于 `micro_batch × grad_accum × num_gpus`。单卡默认 `--batch-size 4` 时 json 中应为 `4`。
>
> **注意 2**：当前 `gpipe` / `1f1b` 在单卡上均为**分段前向仿真**，气泡率由公式 `(K-1)/(M+K-1)` 计算；两者吞吐可能略有差异，但调度逻辑尚未做严格 GPipe / 1F1B 时间线区分。

### 一键运行核心实验

```bash
# 1. 覆盖写入 metrics.csv（baseline + zero1/2/3 + gpipe + 1f1b 实测）
bash scripts/run_all.sh

# 2. 追加多卡 ZeRO 理论值到 metrics.csv
python src/theory.py

# 可用环境变量调参
STEPS=30 BATCH_SIZE=4 bash scripts/run_all.sh
```

`run_all.sh` 启动时会**清空** `results/metrics.csv` 仅保留表头；`theory.py` / `sweep.py` / `combo.py` 及单次 `train.py` 则为**追加**写入各自 CSV。

如果希望避免理论值或扩展数据重复追加，建议使用：

```bash
bash scripts/deploy.sh core
bash scripts/deploy.sh extended
```

其中 `core` 会先通过 `run_all.sh` 覆盖 `metrics.csv`，再追加一次理论行；`extended` 会先重置 `sweep.csv / combo.csv / tradeoff.csv` 表头，再重新生成扩展数据。

### 扩展实验

```bash
# 气泡率 / 模型规模扫描 → results/sweep.csv
python src/sweep.py
python src/sweep.py --sweep bubble    # 仅气泡率扫描
python src/sweep.py --sweep scale     # 仅规模扫描

# PP×ZeRO 组合显存 + 气泡-吞吐权衡 → combo.csv / tradeoff.csv
python src/combo.py
```

部署脚本版本：

```bash
bash scripts/deploy.sh extended
```

> 注意：`src/sweep.py` 与 `src/combo.py` 单独运行时采用追加写入；`deploy.sh extended` 会先覆盖扩展 CSV 表头，适合复现实验和交付前刷新数据。

### 可视化系统

可视化系统位于 `viz/index.html`，读取 `results/` 下四份 CSV，并提供 6 张图、PNG 导出、图1 GPU 数滑块、图2 显存随 M 下降子图，以及 `P / N / ZeRO stage / M` 交互式实验参数面板。

启动方式：

```bash
bash scripts/deploy.sh viz
```

默认访问地址：

```text
http://127.0.0.1:8765/viz/index.html
```

如果端口冲突：

```bash
PORT=9000 bash scripts/deploy.sh viz
```

再访问：

```text
http://127.0.0.1:9000/viz/index.html
```

也可以不用脚本，直接在项目根目录启动静态服务：

```bash
python -m http.server 8765 --bind 127.0.0.1
```

> 不建议直接双击打开 `viz/index.html`。浏览器的本地文件安全策略通常会阻止页面通过 `fetch` 读取 `results/*.csv`。

## 五、接口说明

### 5.1 数据接口：`results/metrics.csv`

平台与下游（可视化、报告）之间的**唯一主数据接口**。列定义固定如下：

| 列名 | 含义 | 取值示例 |
|------|------|---------|
| `config` | 实验配置名 | `baseline` / `zero1` / `zero2` / `zero3` / `gpipe` / `1f1b` / `*_theory` |
| `category` | 实验类别 | `zero`（ZeRO 实测）/ `pipeline`（流水线实测）/ `zero_theory`（多卡理论值） |
| `stage` | ZeRO 阶段 | `0` / `1` / `2` / `3`（流水线为空） |
| `num_gpus` | 卡数 | `1` / `4` / `8` / `16` / `64` |
| `micro_batches` | 微批次数 M | `8`（仅流水线实验） |
| `mem_per_gpu_gb` | 每卡峰值显存(GB) | `2.6836` |
| `throughput_samples_s` | 吞吐(samples/s) | `73.33` |
| `bubble_ratio` | 气泡率(0–1) | `0.2727`（仅流水线实验） |
| `comm_volume` | 通信量(以 Ψ 为单位) | `2`（ZeRO-1/2）/ `3`（ZeRO-3） |

**约定**：
- 列名、单位固定，不得变更（下游已依赖）。
- 缺失值为空字符串。
- **写入策略**：
  - `bash scripts/run_all.sh` → **覆盖** `metrics.csv`（先清空再写实测行）。
  - `python src/train.py` / `python src/theory.py` → **追加**一行。
  - `bash scripts/deploy.sh core` → 覆盖实测行后追加一次理论行，适合交付前复现。
- `category` 区分实测与理论：`zero` / `pipeline` 为单卡真实测量，`zero_theory` 为多卡公式推算（基于 ~30M 参数模型，K=12）。

### 5.2 扩展数据接口

与 `metrics.csv` 分离，供报告 / 可视化深入分析，**不影响下游主接口契约**。

**`results/sweep.csv`**

| 列名 | 含义 |
|------|------|
| `sweep_type` | `bubble` / `scale` |
| `schedule` | `gpipe` / `1f1b`（scale 扫描为空） |
| `num_stages` | 流水线阶段数 K |
| `micro_batches` | 微批次数 M |
| `n_layer` / `n_embd` / `params` | 模型规模 |
| `bubble_theory` / `bubble_measured` / `bubble_abs_err` | 气泡率理论值、记录值、绝对误差 |
| `mem_per_gpu_gb` / `throughput_samples_s` / `step_time_ms` | 显存、吞吐、单 micro-batch 时间 |

**`results/combo.csv`**（实验 A：PP × ZeRO 组合显存）

| 列名 | 含义 |
|------|------|
| `pipeline_stages_P` | 流水线段数 P |
| `zero_stage` | ZeRO 阶段 0–3 |
| `dp_degree_N` | 段内数据并行度 |
| `total_gpus` | P × N |
| `mem_per_gpu_gb` | 每卡模型状态显存(GB) |
| `vs_baseline_x` | 相对 baseline 的节省倍数 |

**`results/tradeoff.csv`**（实验 B：气泡-吞吐权衡）

| 列名 | 含义 |
|------|------|
| `num_stages_K` / `micro_batches_M` | 网格参数 |
| `bubble_ratio` | 理论气泡率 |
| `effective_compute_pct` | 有效算力占比 `(1 - bubble) × 100` |

以上扩展 CSV 在 `src/sweep.py` / `src/combo.py` 单独运行时为**追加**写入；`bash scripts/deploy.sh extended` 会先重置表头再生成，适合避免重复行。

### 5.3 训练入口：`src/train.py`

| 参数 | 默认 | 说明 |
|------|------|------|
| `--config` | `baseline` | 实验类型：baseline / zero1 / zero2 / zero3 / gpipe / 1f1b |
| `--steps` | 10 | 训练步数 |
| `--warmup-steps` | 2 | 预热步数（不计入指标，排除冷启动噪声） |
| `--batch-size` | 4 | 批大小 |
| `--block-size` | 128 | 序列长度 |
| `--n-layer` | 6 | Transformer 层数 |
| `--n-embd` | 384 | 隐藏维度 |
| `--num-stages` | 4 | 流水线阶段数 K |
| `--micro-batches` | 8 | 流水线微批次数 M |
| `--no-export` | — | 不写入 metrics.csv（调试用） |
| `--deepspeed` | — | DeepSpeed 启动开关（ZeRO 实验需配合 `deepspeed` 启动器） |

ZeRO 实验映射关系（`train.py` 内 `ZERO_CONFIGS`）：

| `--config` | json 文件 | stage |
|------------|-----------|-------|
| `zero1` | `configs/zero1.json` | 1 |
| `zero2` | `configs/zero2.json` | 2 |
| `zero3` | `configs/zero3.json` | 3 |

### 5.4 核心模块接口

| 模块 | 关键函数 | 说明 |
|------|---------|------|
| `model.py` | `build_model(...)` | 构建 GPT 模型 |
| `profiler.py` | `bubble_ratio_formula(K, M)` | 气泡率公式 `(K-1)/(M+K-1)` |
| `profiler.py` | `Profiler.summary()` | 返回 `{mem_per_gpu_gb, throughput_samples_s}` |
| `pipeline.py` | `PipelineSimulator.run_steps(...)` | 流水线分段仿真 |
| `theory.py` | `model_state_gb(psi, stage, n)` | ZeRO 各阶段每卡模型状态显存(GB) |
| `sweep.py` | `sweep_bubble` / `sweep_scale` | 参数扫描实验 |
| `combo.py` | `exp_a_combo` / `exp_b_tradeoff` | 组合显存与权衡前沿 |
| `export.py` | `append_row(row)` | 追加一行到 metrics.csv |

## 六、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `wsl` 报 `E_UNEXPECTED` | 安全软件拦截 / WSL 升级残留 | 放行 WSL 网卡；重装 WSL MSI；配置 `.wslconfig` 镜像网络 |
| `CUDA_HOME does not exist` | 缺 CUDA Toolkit | 通常可忽略（已用 `torch_adam`）；若仍报错则 `conda install cuda-toolkit` 并设 `CUDA_HOME` |
| `train.py: error: unrecognized arguments` | `--deepspeed` 误传了 json 路径 | `--deepspeed` 是无值开关；json 由 `--config` 自动加载 |
| DeepSpeed batch size 校验失败 | `train_batch_size` 与实际不符 | 改 json 使 `train_batch_size = batch × grad_accum × num_gpus` |
