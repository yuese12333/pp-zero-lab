# PP-ZeRO 可视化系统说明文档

## 完成内容

本目录实现了 `给B的交付说明.md` 中要求的可视化系统，入口文件为 `index.html`。页面会读取项目根目录 `results/` 下的四份 CSV：

- `results/metrics.csv`
- `results/sweep.csv`
- `results/combo.csv`
- `results/tradeoff.csv`

页面按列名解析 CSV，不依赖行顺序，并在加载时检查关键列是否存在。

## 额外加分项

已完成第四部分的两个加分项：

1. 图2 旁边增加了“显存随 M 下降”子图。
   - 数据来源：`sweep.csv` 中 `sweep_type == bubble` 且 `schedule == gpipe` 的行。
   - 使用字段：`micro_batches` 与 `mem_per_gpu_gb`。
   - 展示结论：M 从 1 增至 32 时，每卡显存约从 4.07GB 降到 0.70GB，说明 micro-batch 越小，激活显存越低。

2. 图1 增加了 `num_gpus` 交互滑块。
   - 可选 GPU 数：1、4、8、16、64。
   - 拖动滑块会动态更新图1中的竖向参考线、高亮点和文字指标。
   - 指标区会同步显示当前 N 下 Baseline、ZeRO-1、ZeRO-2、ZeRO-3 的每卡显存。

## 交互式实验参数面板

已基于现有数据增加真实交互式实验参数面板。该面板不编造新数据，而是直接查表使用现有 CSV：

- `combo.csv`：提供 `P / N / ZeRO stage` 对应的每卡模型状态显存、总 GPU 数和相对 Baseline 压缩倍数。
- `tradeoff.csv`：提供 `K / M` 对应的气泡率和有效算力。

面板中的 `P / K` 共用同一个选择框：

- 在显存计算中表示 `pipeline_stages_P`。
- 在气泡率计算中表示 `num_stages_K`。

当前可选范围由 CSV 自动读取：

- `P`：1、2、4、8
- `N`：1、4、8
- `ZeRO stage`：0、1、2、3
- `M`：1、2、4、8、16、32、64

注意：`tradeoff.csv` 的 K 支持 2、4、8、16，而 `combo.csv` 的 P 支持 1、2、4、8。因此当选择 `P=1` 时，显存指标可以计算，但气泡率和有效算力没有直接数据，页面会给出提示。类似地，Baseline / ZeRO-0 不做数据并行分片，`combo.csv` 中只保留 `N=1`，选择 `ZeRO-0` 且 `N=4/8` 时页面也会提示无直接组合。

## 启动方式

由于浏览器直接打开本地 HTML 时通常不允许 `fetch` 读取相邻 CSV 文件，建议从项目根目录启动一个本地 HTTP 服务。

在 PowerShell 中进入项目根目录：

```powershell
cd .\pp-zero-lab
python -m http.server 8765 --bind 127.0.0.1
```

然后在浏览器中打开：

```text
http://127.0.0.1:8765/viz/index.html
```

如果 8765 端口已被占用，可以换成其他端口，例如：

```powershell
python -m http.server 9000 --bind 127.0.0.1
```

对应访问地址改为：

```text
http://127.0.0.1:9000/viz/index.html
```

## 使用说明

- 点击 `刷新数据` 可以重新读取 CSV 并刷新全部图表。
- 在“交互式实验参数面板”中选择 `P / N / ZeRO stage / M`，可以动态查看每卡显存、压缩倍数、总 GPU 数、气泡率和有效算力。
- 拖动图1下方的 GPU 数滑块，可以查看不同 `num_gpus` 下的 ZeRO 显存变化。
- 点击单张图右上角的 `PNG` 可以导出该图。
- 点击页面顶部的 `导出全部 PNG` 可以依次导出 6 张图。
